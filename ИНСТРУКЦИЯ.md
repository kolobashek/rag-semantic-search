# 📚 RAG Каталог - Полная Инструкция

## Обзор системы

RAG система состоит из:
1. **Индексер** (index_rag.py) - индексирует DOCX/XLSX/PDF файлы
2. **Поиск** (rag_search.py) - семантический поиск (CLI)
3. **OCR** (ocr_pdfs.py) - второй проход: OCR для сканированных PDF
4. **Web UI** (app_ui.py) - Streamlit интерфейс в браузере
5. **Windows App** (windows_app.py) - Native Windows приложение
6. **EXE Builder** (build_exe.bat) - сборка в standalone exe
7. **Ядро** (rag_core.py) - общая логика подключения/поиска (Qdrant + embeddings)

## Быстрый старт

### 1. Установка зависимостей

```powershell
pip install python-docx openpyxl xlrd pdfplumber qdrant-client sentence-transformers PyQt6 PyInstaller streamlit tqdm
```

### 2. Индексирование (первый раз, 2-3 часа)

```powershell
cd O:\Обмен
python index_rag.py --recreate --catalog "O:\Обмен"
```

Проверьте логи:
- "Indexation complete! Total points: 342,856"
- Создана папка D:\qdrant_state

### 3. Тестируйте поиск

#### Вариант A: Streamlit (в браузере)
```powershell
streamlit run app_ui.py
# Откроется http://localhost:8501
```

#### Вариант B: Windows приложение
```powershell
python windows_app.py
# Откроется native Windows окно
```

#### Вариант C: CLI (командная строка)
```powershell
python rag_search.py --query "договоры газпром" --limit 10
```

### 4. Соберите EXE

```powershell
.\build_exe.bat
# Результат: dist\RAG-Katalog.exe (~300 MB)
```

## Использование индексера

### Полная переиндексация (первый раз)
```powershell
python index_rag.py --recreate
```

### Обновление (только новые файлы)
```powershell
python index_rag.py
# Займет 5-10 минут вместо 2-3 часов!
```

### С пользовательскими параметрами
```powershell
python index_rag.py --catalog "C:\MyDocuments" --db "C:\my_rag_db" --recreate
```

**Опции:**
- `--catalog PATH` - папка для индексирования
- `--url URL` - URL Docker Qdrant-сервера (например `http://localhost:6333`); приоритет над `--db`
- `--db PATH` - папка для локальной Qdrant БД (если `--url` не задан)
- `--model NAME` - модель embeddings
- `--collection NAME` - имя коллекции
- `--workers N` - число рабочих потоков (2 рекомендуется; 1 — OCR-проход)
- `--recreate` - пересоздать коллекцию с нуля (медленно)
- `--cleanup` - только удалить из индекса файлы, которых нет на диске (~1 мин)
- `--no-ocr` - пропускать OCR для сканированных PDF (быстрый первый проход)
- `--max-chunks N` - максимум чанков с одного файла (по умолчанию 2000; 0 = без ограничений)

### Рекомендуемый двухэтапный запуск

```powershell
# Этап 1 — быстрый проход: DOCX/XLSX/PDF с текстом (без OCR)
#   Используйте Docker Qdrant (localhost:6333) для максимальной скорости
python index_rag.py --no-ocr --url http://localhost:6333 --workers 2

# Этап 2 — OCR для сканированных PDF (требует Tesseract + Poppler в PATH)
#   ocr_pdfs.py автоматически находит PDF с пустым текстом и переиндексирует их
python ocr_pdfs.py --url http://localhost:6333
```

### Установка Poppler (для OCR)

Portable-версия без прав администратора:
```powershell
# Скачать и распаковать
$url = "https://github.com/oschwartz10612/poppler-windows/releases/download/v24.08.0-0/Release-24.08.0-0.zip"
Invoke-WebRequest -Uri $url -OutFile poppler.zip -UseBasicParsing
Expand-Archive poppler.zip C:\Users\$env:USERNAME\poppler -Force

# Добавить в PATH (текущая сессия)
$env:PATH = "C:\Users\$env:USERNAME\poppler\poppler-24.08.0\Library\bin;" + $env:PATH
```

Или использовать `run_index.py` — он добавляет Poppler автоматически.

### Автоматизация (run_automation.py)

```powershell
# Инкрементальное обновление (по умолчанию, 5-10 мин)
python run_automation.py

# Полная переиндексация (2-3 часа)
python run_automation.py --recreate
```

## Установка OCR для сканированных PDF

Для распознавания PDF без текстового слоя нужны дополнительные компоненты.

### 1. Установить Python-библиотеки

```powershell
pip install pytesseract pdf2image
```

### 2. Установить Tesseract (движок OCR)

Скачать установщик для Windows:
https://github.com/UB-Mannheim/tesseract/wiki

При установке отметить языковой пакет **Russian**.

Убедиться, что путь к `tesseract.exe` добавлен в системный PATH.

### 3. Установить Poppler (конвертация PDF → изображение)

**Способ A — через Chocolatey (требует прав администратора):**
```powershell
choco install poppler
```

**Способ B — вручную (без прав администратора):**
1. Скачать portable-версию: https://github.com/oschwartz10612/poppler-windows/releases
2. Распаковать, например, в `C:\poppler`
3. Добавить `C:\poppler\Library\bin` в системный PATH

### Проверка

```powershell
tesseract --version
pdftoppm -v
```

Если оба вывели версию — OCR работает.

**Возможности:**

- 💾 Preview текста в результатах

**Примеры запросов:**
- "договоры"
- "паспорта сотрудников"
- "счета газпром"
- "служебная записка"

### Windows приложение (PyQt6)

```powershell
python windows_app.py
```

Или запустите скомпилированный EXE:
```powershell
dist\RAG-Katalog.exe
```

**Возможности:**
- Фоновый поиск (не блокирует UI)
- Скроллируемые результаты
- Быстрые кнопки поиска
- Статус подключения
- Progress bar

### CLI интерфейс (командная строка)

#### Интерактивный режим
```powershell
python rag_search.py
# Введите Запрос: договоры
# Результаты в консоли
# Введите 'help' для справки по фильтрам
```

#### Прямой поиск с параметрами
```powershell
# Базовый поиск (Docker Qdrant)
python rag_search.py --url http://localhost:6333 --query "договоры газпром"

# С фильтрами
python rag_search.py --query "счета" --type .xlsx --limit 5

# JSON вывод
python rag_search.py --query "паспорта" --limit 10 > results.json

# Только содержимое (без метаданных файлов)
python rag_search.py --query "служебная записка" --content-only
```

**Опции:**
- `--query TEXT` - поисковый запрос (без — интерактивный режим)
- `--url URL` - URL Docker Qdrant-сервера (по умолчанию из config.json)
- `--limit N` - количество результатов (по умолчанию 10)
- `--type EXT` - фильтр по расширению (.docx, .xlsx, .pdf)
- `--content-only` - исключить метаданные файлов
- `--db PATH` - папка локальной Qdrant БД (если `--url` не задан)
- `--collection NAME` - имя коллекции

**Встроенные флаги (интерактивный режим):**
```
/type .docx договоры — фильтр по типу файла
/content паспорта   — только содержимое
```

## Производительность

### Индексирование

| Операция | Время | Комментарий |
|----------|-------|-----------|
| Первое индексирование | 2-3 часа | Все 31K файлов |
| Обновление (50 новых) | 5-10 мин | Incremental indexing |
| Проверка (нет изменений) | <1 мин | Только проверка state |

### Поиск

| Операция | Время | Комментарий |
|----------|-------|-----------|
| Один запрос | 0.5-2 сек | Локальный Qdrant |
| Загрузка UI | 3-5 сек | Первый запуск модели |
| EXE запуск | 5-10 сек | Загрузка всех зависимостей |

### Используемые ресурсы

| Ресурс | Объем | Комментарий |
|--------|-------|-----------|
| Qdrant БД | 10-15 GB | 342K indexed points |
| EXE файл | 200-300 MB | Со всеми зависимостями |
| RAM при поиске | 2-3 GB | Модель embeddings в памяти |

## Решение проблем

### Проблема: "Qdrant not found" / "Connection refused"

**Решение (Docker Qdrant):**
```powershell
# 1. Убедитесь что Docker Desktop запущен
# 2. Проверьте что контейнер работает
docker ps | findstr qdrant

# 3. Если контейнер не запущен — запустите
docker run -d -p 6333:6333 -v D:\qdrant_data:/qdrant/storage qdrant/qdrant

# 4. Проверьте коллекцию
python -c "from qdrant_client import QdrantClient; qd = QdrantClient(url='http://localhost:6333'); print(qd.get_collections())"

# 5. Если коллекции нет — переиндексируйте
python index_rag.py --url http://localhost:6333 --recreate
```

### Проблема: "No module named sentence_transformers"

**Решение:**
```powershell
pip install --upgrade sentence-transformers
pip install --upgrade qdrant-client
```

### Проблема: Зависание на старте (Python 3.14)

**Симптом:** `index_rag.py` или поиск зависает на несколько минут при запуске.

**Причина:** Python 3.14 вызывает `platform._wmi_query()` при импорте torch, что зависает на некоторых Windows-машинах.

**Решение:** Патч централизован в `_platform_compat.py` и вызывается в entrypoint до тяжёлых импортов.
Если зависание всё равно происходит — убедитесь, что вызов `apply_windows_platform_workarounds()` выполняется до `sentence_transformers/torch` импортов.

### Проблема: EXE не запускается

**Решение:**
```powershell
# Запустите из консоли чтобы увидеть ошибку
cd dist
.\RAG-Katalog.exe

# Или запустите Python версию напрямую
python windows_app.py
```

### Проблема: Очень долгое первое индексирование

**Решение:**
- Это нормально (2-3 часа на 31K файлов)
- Проверьте что кодировка файлов правильная
- Убедитесь что диск не перегружен
- Можно запустить в фоне:
  ```powershell
  Start-Process python -ArgumentList "index_rag.py --recreate" -NoNewWindow
  ```

## Оптимизация

### Увеличить скорость поиска

1. Уменьшить chunk_size в index_rag.py (по умолчанию 500)
2. Использовать более быструю модель embeddings (хотя качество упадет)

### Уменьшить размер EXE

```powershell
# Используйте --onedir вместо --onefile
pyinstaller --name="RAG-Katalog" --onedir --windowed windows_app.py

# Результат: отдельные файлы в папке, но быстрее загружается
```

### Уменьшить использование RAM

```powershell
# Уменьшить batch size в index_rag.py (BATCH_SIZE = 500 вместо 1000)
# Или использовать меньше overlapping chunks (CHUNK_OVERLAP = 50)
```
