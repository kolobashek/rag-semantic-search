# 📚 Технопоиск — Инструкция по эксплуатации

## Обзор системы

| Компонент | Файл | Назначение |
|---|---|---|
| Лаунчер | `rag_catalog.cli.launcher` | Единый запуск web + Qdrant + бот |
| Web UI | `nice_app.py` | Основной интерфейс (NiceGUI, `localhost:8080`) |
| Индексер | `index_rag.py` | Индексирование DOCX/XLSX/PDF (3-этапный) |
| OCR-проход | `ocr_pdfs.py` | Второй проход: OCR для сканированных PDF |
| CLI-поиск | `rag_search.py` | Семантический поиск из командной строки |
| Telegram-бот | `telegram_bot.py` | Поиск через Telegram |
| Legacy Web UI | `app_ui.py` | Streamlit-интерфейс (устарел, оставлен для совместимости) |
| Windows App | `windows_app.py` | Native PyQt6-приложение |

Хранилище векторов — Qdrant. Модель эмбеддингов — `sentence-transformers`.
Конфигурация — `config.json` (шаблон: `config.example.json`).
Рабочие данные и БД хранятся в папке `data/` внутри проекта (не в `D:\qdrant_state`).

## Быстрый старт

### 1. Установка зависимостей

```powershell
pip install -r requirements.txt
```

### 2. Конфигурация

```powershell
Copy-Item config.example.json config.json
# Отредактируйте config.json: укажите catalog_path, qdrant_url и т.д.
```

### 3. Запуск через лаунчер (рекомендуется)

```powershell
python -m rag_catalog.cli.launcher start
# Статус
python -m rag_catalog.cli.launcher status
# Остановка
python -m rag_catalog.cli.launcher stop
```

Лаунчер поднимает:
- **Web UI** на `http://127.0.0.1:8080`
- **Qdrant** (локальный Docker) — только если `qdrant_url` указывает на localhost и порт недоступен
- **Telegram-бот** — только если `telegram_enabled=true` и токен задан

### 4. Первичное индексирование

Рекомендуемый трёхэтапный запуск (быстро → полно → OCR):

```powershell
# Этап 1: только метаданные всех файлов (минуты)
python index_rag.py --stage metadata

# Этап 2: полное содержимое DOCX/XLSX и небольших PDF (десятки минут)
python index_rag.py --stage small

# Этап 3: большие файлы + сканированные PDF с OCR (часы)
python index_rag.py --stage large
```

Или все этапы сразу:

```powershell
python index_rag.py --stage all
```

После `--stage metadata` поиск по именам файлов уже работает; `small` и `large` добавляют полнотекстовый поиск.

### 5. Открыть интерфейс

Откройте в браузере: `http://localhost:8080`

## Индексирование

### Опции `index_rag.py`

```
--stage [metadata|small|large|all]   Этап индексирования (default: all)
--catalog PATH                       Папка каталога (default: из config.json)
--url URL                            URL Qdrant-сервера (приоритет над --db)
--db PATH                            Папка локальной Qdrant БД (если нет --url)
--model NAME                         Модель эмбеддингов
--collection NAME                    Имя коллекции
--workers N                          Число рабочих потоков (рекомендуется 2–4)
--recreate                           Пересоздать коллекцию и очистить state
--cleanup                            Удалить из индекса файлы, которых нет на диске
--no-ocr                             Пропустить OCR в текущем проходе
--max-chunks N                       Максимум чанков с одного файла (default: 2000)
--onnx                               Использовать ONNX-ускорение эмбеддингов
--mark-stage-metadata-for EXT        Сброс stage→metadata для расширения (миграция legacy)
```

### Этапы индексирования

| Этап | Что индексируется | Время | Назначение |
|---|---|---|---|
| `metadata` | Имя, путь, размер, mtime | Минуты | Поиск по именам сразу |
| `small` | DOCX, XLSX, XLS, PDF ≤ порога | Десятки минут | Основной полнотекстовый поиск |
| `large` | Большие PDF, сканы (OCR) | Часы | Полное покрытие |
| `all` | Все три этапа подряд | Часы | Первичная полная индексация |

### Инкрементальное обновление

```powershell
# Обновить только новые/изменённые файлы (без --recreate)
python index_rag.py --stage all
# ~5–10 минут вместо нескольких часов для небольших обновлений
```

### Очистка индекса

```powershell
# Удалить из индекса файлы, которых нет на диске
python index_rag.py --cleanup
```

### Полная переиндексация

```powershell
python index_rag.py --recreate --stage all
```

### Миграция legacy state (однократно)

Если ранее использовался старый `--metadata-only-for .pdf`, нужно сбросить stage для PDF:

```powershell
python index_rag.py --mark-stage-metadata-for .pdf
python index_rag.py --stage large
```

## OCR для сканированных PDF

### Bundled-инструменты (рекомендуется)

Поместите portable-бинарники в папку `tools/` проекта — глобальный PATH не нужен:

```
tools/tesseract/tesseract.exe   (или tools/tesseract/bin/tesseract.exe)
tools/poppler/Library/bin/      (или tools/poppler/bin/)
```

Можно переопределить пути явно в `config.json`:
- `ocr_tesseract_cmd` — абсолютный путь к `tesseract.exe`
- `ocr_poppler_bin` — абсолютный путь к папке `bin` Poppler

Или через переменные окружения: `RAG_TESSERACT_CMD`, `RAG_POPPLER_BIN`.

### Установка Tesseract (если нет bundled)

Скачать: https://github.com/UB-Mannheim/tesseract/wiki  
При установке выбрать языковой пакет **Russian**.  
Добавить путь к `tesseract.exe` в PATH.

### Установка Poppler (без прав администратора)

```powershell
$url = "https://github.com/oschwartz10612/poppler-windows/releases/download/v24.08.0-0/Release-24.08.0-0.zip"
Invoke-WebRequest -Uri $url -OutFile poppler.zip -UseBasicParsing
Expand-Archive poppler.zip C:\Users\$env:USERNAME\poppler -Force
$env:PATH = "C:\Users\$env:USERNAME\poppler\poppler-24.08.0\Library\bin;" + $env:PATH
```

### Проверка OCR

```powershell
tesseract --version
pdftoppm -v
```

### OCR-проход поверх уже проиндексированного каталога

```powershell
python ocr_pdfs.py --url http://localhost:6333
```

## Cloud Drive

Cloud Drive — встроенный файловый реестр с web-интерфейсом и storage backend.

### Инициализация

```powershell
python cloud_drive.py init --enable
python cloud_drive.py stats
python cloud_drive.py bootstrap --max-files 1000
```

### Настройки в `config.json`

```json
{
  "cloud_drive_enabled": true,
  "cloud_drive_db_path": "data/cloud_drive.db",
  "cloud_drive_storage": "local",
  "cloud_drive_storage_root": "data/cloud_storage",
  "cloud_drive_bucket": ""
}
```

Для S3/MinIO дополнительно: `cloud_drive_s3_endpoint`, `cloud_drive_s3_region`, `cloud_drive_s3_access_key`, `cloud_drive_s3_secret_key`.

### Доступные API-эндпоинты

```
GET  /api/cloud-drive/node           — узел реестра по path
GET  /api/cloud-drive/list           — список дочерних узлов
POST /api/cloud-drive/folders        — создать папку
POST /api/cloud-drive/upload         — загрузить файл
GET  /api/cloud-drive/download       — скачать файл
GET  /api/cloud-drive/versions       — версии файла
POST /api/cloud-drive/move           — переместить
POST /api/cloud-drive/rename         — переименовать
POST /api/cloud-drive/delete         — удалить
POST /api/cloud-drive/reindex        — поставить в очередь на переиндексацию
GET  /api/cloud-drive/jobs           — список фоновых задач
GET  /api/cloud-drive/job            — статус задачи по ID
GET  /api/cloud-drive/job-latest     — последняя задача по типу
GET  /api/cloud-drive/bootstrap-status    — статус bootstrap
GET  /api/cloud-drive/bootstrap-jobs      — история bootstrap
GET  /api/cloud-drive/storage-health      — healthcheck хранилища
```

### Admin UI

`Настройки → Cloud Drive`:
- Инициализировать реестр, Статистика, Импортировать структуру
- Мониторинг фоновых задач с прогрессом, cancel/retry

## Поиск

### Web UI (основной)

`http://localhost:8080` — семантический поиск, проводник Cloud Drive, статистика, настройки.

### CLI

```powershell
# Интерактивный режим
python rag_search.py

# Прямой запрос
python rag_search.py --query "договоры газпром" --limit 10

# С фильтром по типу
python rag_search.py --query "счета" --type .xlsx --limit 5

# Через Docker Qdrant
python rag_search.py --url http://localhost:6333 --query "паспорта сотрудников"
```

**Опции:**
```
--query TEXT        Поисковый запрос
--url URL           URL Qdrant-сервера
--limit N           Число результатов (default: 10)
--type EXT          Фильтр: .docx, .xlsx, .pdf
--content-only      Исключить метаданные файлов
--db PATH           Локальная Qdrant БД
--collection NAME   Имя коллекции
```

### Telegram-бот

Запустить через лаунчер или напрямую:

```powershell
python telegram_bot.py
```

## Решение проблем

### Qdrant недоступен

```powershell
# Проверить контейнер
docker ps | findstr qdrant
# Запустить, если не запущен
docker run -d -p 6333:6333 -v D:\qdrant_data:/qdrant/storage qdrant/qdrant
# Проверить коллекцию
python -c "from qdrant_client import QdrantClient; c = QdrantClient(url='http://localhost:6333'); print(c.get_collections())"
```

### ModuleNotFoundError

```powershell
pip install --upgrade sentence-transformers qdrant-client
```

### Зависание при старте (Python 3.14)

Причина: `platform._wmi_query()` зависает при импорте torch на некоторых Windows-машинах.  
Исправление централизовано в `_platform_compat.py` — вызов `apply_windows_platform_workarounds()` должен стоять до тяжёлых импортов.

### Где хранятся рабочие данные

| Что | Где |
|---|---|
| Qdrant state, index_state, telemetry, users DB | `data/` (в корне проекта) |
| Launcher PID / runtime state | `runtime/` |
| Логи | `logs/` |

Эти папки не коммитятся в Git.
